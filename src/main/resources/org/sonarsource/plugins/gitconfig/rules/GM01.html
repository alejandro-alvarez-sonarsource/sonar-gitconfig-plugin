Why is this an issue?

Git supports six different protocols:

  1. Local protocol for removes starting by / or file://
  2. git://
  3. Plain http://
  4. Git over SSH ssh://
  5. Secure https://

The first protocol to a local mirror, which can be unavailable to third parties cloning the repository.

The second and the third protocols are insecure -- not encrypted -- and can be hijacked by an attacker.

Only the third and the fourth allow third parties to securely clone the submodule. However, the ssh protocol requires either a password or a private key known by anyone wanting to clone the submodule. If the project is public, this can be inconvenient for users.

Hence, https:// is to be preferred.
